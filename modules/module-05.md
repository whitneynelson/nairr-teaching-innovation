*Feeds: Day 3, 2:30–3:00 PM*

A short session; it's fine to set the direction here and finish the details independently, especially the AI-use rubric, which deserves real thought.

> *"Draft a grading scale and category weighting for a course with these assessment types: [list from your syllabus/curriculum map]."*

Fill in `course-toolkit/grading-scale.md`.

## Step 2: Build the assignment rubric template

`rubrics/assignment-rubric-template.md`; a reusable rubric shape, not a rubric for one specific assignment.

## Step 3: Build the AI-use rubric

Since assignments involve Claude either writing code (coders) or generating it from plain-language descriptions (non-coders), your rubric needs criteria that make sense for both:

- Did the student disclose how they used Claude?
- Coders: did they understand and can they explain the code Claude helped write?
- Non-coders: can they explain what the generated script does, even if they didn't write it line-by-line?
- Any academic integrity flags specific to your policy
